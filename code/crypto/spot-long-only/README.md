# Crypto Spot Long-Only Code Appendix

This appendix contains the scripts and modules behind the public Crypto active baseline.

## Can This Run?

| Component | Public runnable? | Data needed |
|---|---|---|
| `research-scripts/02_spot_universe_baseline.py` | Yes | Internet access for Binance daily spot klines |
| `research-scripts/10_final_predeployment_validation.py` | Yes after baseline data exists | Binance data generated by baseline script |
| `research-scripts/01-11` | Mostly yes, but script sequence matters | Generated intermediate CSVs from earlier scripts |
| `spot_trend_core/` | Partially | User-supplied OHLCV cache following the documented schema |

## Public Role

The strategy is a spot-only, long-only trend sleeve:

```text
entry: close above prior 20-day high and above prior EMA200
exit: close-based 3ATR trailing exit
sizing: volatility-scaled at entry
cash: idle when no sleeve signal exists
shorting: prohibited
per-symbol tuning: prohibited
```

## Data Boundary

The public research scripts fetch Binance spot daily klines and may cache generated OHLCV files under:

```text
code/crypto/spot-long-only/spot_data_cache/
```

That cache is ignored by `.gitignore` and should not be committed.
