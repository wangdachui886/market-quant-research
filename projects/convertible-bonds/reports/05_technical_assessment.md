# 05_technical_assessment

## 数据与未来函数

- 正股动量使用每只正股历史价格的 20 日变化，信号日收盘后观察，T+2 执行。
- 流动性试验曾使用 shifted trailing amount，避免使用调仓日成交额作为当日入池条件。
- 强赎过滤使用 notice_date 的 PIT 表；公告日及之后不再保留。
- conversion price 缺失只做同一转债内 forward-fill，不做 backward-fill；这符合“已知转股价继续有效”的口径。

## 工程风险

- 中文路径、编码和 Windows shell 是主要操作风险，脚本均尽量使用 UTF-8 和绝对路径。
- 实盘前必须锁定数据供应商口径，尤其是强赎公告、停牌、退市期、转股价变更。
- 当前 core 是 pre-live 核心，不是交易所直连系统；下单、风控、对账仍需人工或单独交易执行层。

## 已知弱点

- long-only beta 暴露无法完全消除。
- 2022/2023 类似环境仍可能出现 15%-25% 回撤。
- 调仓日偏移不全胜，说明路径敏感性仍在。
- 策略容量主要适合偏小资金，资金越大越依赖真实流动性和滑点控制。
