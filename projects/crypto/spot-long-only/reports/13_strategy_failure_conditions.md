# 右侧现货 long-only：失效条件与复盘警示

生成时间：2026-05-27

## 1. 定位

本文件不是悲观免责声明，而是实盘后的复盘治理规则。

策略未来表现变差时，不能简单用“最近亏了”判断失效，也不能因为短期回撤就重新调参数。更合理的判断方式是：

> 只有当策略的核心 alpha 来源被持续破坏，才认为策略需要降级、暂停或重新研究。

当前策略的 alpha 来源已经定义为：

1. 趋势确认后参与 crypto 主流币的阶段性右尾；
2. 无趋势时留现金，降低错误行情风险暴露；
3. 多币趋势启动和结束不同步，降低单一 BTC 路径依赖；
4. ATR 收盘确认退出，在强趋势中保留空间，在趋势破坏后退出。

因此，失效条件也应围绕这四个来源来判断。

## 2. 不是失效的情况

以下现象单独出现时，不应直接判定策略失效：

- 牛市主升段跑输满仓 Buy & Hold；
- 某一年收益低、Calmar 低或空仓时间长；
- 某几个币没有贡献，或者短期拖累组合；
- 策略在急涨初期进场较慢；
- 因为收盘确认退出，在盘中暴跌或跳空中没有最优成交；
- 静态 B&H 原始 CAGR 或终值更高。

这些都是策略结构的自然代价。当前策略本来就不是满仓 beta 替代品，而是风险效率 alpha。

## 3. 主要失效来源

### 3.1 趋势延续性消失

如果 crypto 主流币从“阶段性右尾趋势”转为长期高噪音、低延续、快速均值回复市场，20 日突破 + EMA200 的右侧确认会失去意义。

观察信号：

- 突破后 20-60 日 forward return 长期接近 0 或为负；
- 假突破次数持续上升；
- 入场后很快触发 ATR 退出，持仓周期显著缩短；
- 胜率下降的同时，盈亏比也下降。

这是最核心的 alpha 失效风险。

### 3.2 现金过滤不再改善风险效率

当前策略的重要价值是：没信号时留现金，牺牲部分牛市弹性，换取更低回撤和更高 Sharpe / Calmar。

如果未来现金过滤只降低收益，却不再明显降低回撤，策略定位就会被破坏。

观察信号：

- 同波动 Buy & Hold 在多个滚动窗口里持续优于策略；
- 策略 MDD 不再明显低于同池 B&H；
- 策略平均暴露下降，但 Sharpe / Calmar 没有改善；
- 现金期后的重新入场经常追在局部高点。

这不代表必须立刻改成满仓，而是说明“风险效率 alpha”需要降级复盘。

### 3.3 多币异步性消失

12 币池的解释不是“币之间完全低相关”，而是“趋势信号让收益实现路径不同步”。

如果未来信号高度同进同出，或者最大回撤期同时持仓币数显著上升，分散效果会下降。

观察信号：

- 信号相关性上升，多个币在同一天附近集中开仓和退出；
- 最大回撤期平均持仓币数明显高于历史；
- 组合回撤越来越接近 Core4 或满仓 12 币 B&H；
- BTC 路径重新成为组合主要解释变量。

这会削弱 12 币池相对 BTC only / Core4 的风险效率优势。

### 3.4 ATR 退出与波动结构失配

3ATR 收盘确认退出适合日线趋势跟踪，但它不是万能止损。

未来如果波动结构变化，ATR 退出可能出现两类问题：

- 太紧：频繁被震出，趋势还没结束就退出；
- 太慢：趋势破坏后迟迟退出，回撤扩大。

观察信号：

- 持仓胜率和单笔平均收益同时恶化；
- 退出后价格频繁快速修复，说明被噪音震出；
- 退出前回撤明显扩大，说明止损响应太慢；
- 3ATR trailing exit 的贡献从保护回撤变成系统性拖累。

这类问题只能通过封存后的边界复盘判断，不能在实盘中随意调 ATR 倍数。

### 3.5 Universe 老化或流动性退化

如果币池本身不再代表可交易的主流现货资产，策略信号再好也会被执行和样本质量破坏。

观察信号：

- 某些币交易状态异常、下架、限制交易或长期维护；
- 90D / 30D 成交额持续下降；
- 订单金额占 ADV 比例过高；
- 数据缺 K、重复 K、异常 OHLC 或成交量失真；
- 新主流资产长期被排除，而旧币持续衰退。

这由 Universe health monitor 处理，不能通过回测收益排名主观换币。

### 3.6 执行摩擦吞噬收益

策略是日线现货系统，边际收益不能被手续费、滑点、精度、最小下单额和 API 异常吞掉。

观察信号：

- 实盘成交价长期劣于研究假设；
- 小额订单因最小交易额、精度或余额碎片导致执行偏离；
- 交易所 API、数据延迟、订单失败影响信号执行；
- 实盘净收益与纸面信号长期偏离，且偏离来自执行成本而不是行情。

这属于部署层失效，不一定说明 alpha 失效，但会导致策略不可实盘。

### 3.7 治理失效

最大的非市场风险是把策略越改越像事后优化。

危险动作包括：

- 因短期亏损调 20D、EMA200 或 ATR 倍数；
- 因近期贡献差剔除某个币；
- 因牛市跑输 B&H 而加入底仓或 active redeploy；
- 因近期表现好扩大币池或提高风险；
- 用当前存活币列表回看历史并声称动态池有效。

如果发生这些动作，即使短期收益变好，也会污染策略结论。

## 4. 黄灯与红灯

建议把失效观察分为黄灯和红灯。

### 黄灯

出现以下情况时进入重点观察，但不立刻改策略：

- 12 个月滚动 Sharpe / Calmar 明显下降；
- 假突破次数明显增加；
- 同波动 B&H 开始阶段性胜出；
- 最大回撤期持仓币数上升；
- 执行滑点或订单失败明显增加；
- Universe 中出现 `WATCH` 或 `SUSPENDED` 状态币。

### 红灯

出现以下情况时，应暂停加仓、降级策略状态，进入正式复盘：

- 24-36 个月滚动窗口里，策略不再相对 B&H 改善 MDD / Sharpe / Calmar；
- 同波动 B&H 在多数滚动窗口中持续胜出；
- 最大回撤优势消失，且回撤来源不是单次极端事件；
- 突破后 forward return 长期无效或为负；
- 12 币池相对 BTC only / Core4 的风险效率优势消失；
- 执行成本或交易约束足以吞噬策略边际收益；
- Universe 大面积进入 `WATCH / SUSPENDED`，但没有合格替代池。

红灯不等于立刻废弃策略，但意味着不能继续按原 baseline 自信部署。

## 5. 复盘原则

未来复盘时按以下顺序判断：

1. 先确认是否是正常策略代价，而不是 alpha 失效；
2. 再确认是市场结构问题、Universe 问题、执行问题，还是治理问题；
3. 只有当核心机制被持续破坏时，才进入重新研究；
4. 重新研究先做机制检验，不直接扫参数；
5. 任何新版本必须重新对比 BTC only、Core4、Pool12、Buy & Hold 和同波动 B&H。

## 6. 归档结论

> 这个策略未来最可能的失效原因，不是某一年跑输满仓持有，而是趋势突破延续性下降、现金过滤不再改善风险效率、多币异步性消失、ATR 退出与新波动结构失配、币池老化或执行摩擦上升。
>
> 因此终稿中必须保留失效条件警示。它的作用是防止未来在正常回撤中乱调参数，也防止在核心机制已经坏掉时继续硬撑。
