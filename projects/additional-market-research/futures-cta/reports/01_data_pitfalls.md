# 01 · 数据陷阱：连续合约的"假跳空"与三种处理方法

> 写任何 CTA 回测之前必读。这一节讲清楚为什么免费连续合约数据**不能直接拿来回测**。

---

## 一、问题源头：连续合约不是"一个东西"

### 直觉陷阱

新手看到 Sina/AkShare 提供的"豆粕主力连续"（M0），自然会以为这是一个连续的、可以直接回测的价格序列——就像比特币价格一样。

**这是错的。** M0 实际上是把多个历史主力合约**简单拼接**起来的：

```
M0 价格序列：
  2023-01 ~ 2023-03  ← 来自 M2305 合约
  2023-04 ~ 2023-07  ← 来自 M2309 合约   ← 切换日：价格直接跳!
  2023-08 ~ 2023-12  ← 来自 M2401 合约   ← 切换日：价格直接跳!
  ...
```

切换日没有任何复权处理。如果换月那天 M2305 收盘 3400、M2309 开盘 3500，M0 序列就出现一个 +100 元的"假跳空"。

### 为什么这是大问题

回测系统会把这个 +100 元当成"真实收益"，导致：
- **NAV 失真**：长期累积大量假涨/假跌
- **波动率失真**：跳空导致 vol 估计偏高
- **信号失真**：动量策略可能在假跳空当天被触发

实测案例（M0 豆粕，2017-2025）：
- 不复权 NAV：基本原地踏步（看起来豆粕 8 年没涨）
- 收益率拼接 NAV：**+78%**
- **差额 78% 全部来自假跳空被错误累计**

→ 完整图见 `Phase_2_0_universe/outputs/13_M_clean_vs_raw_nav.png`

## 二、三种处理方法

### 方法 A：不复权连续（Sina/AkShare 默认）

直接拼接，**不可用于回测**。只用于：看品种历史价格区间、做趋势可视化。

### 方法 B：前复权（forward-adjusted）

历史价格**整体平移**，对齐到最新合约价格。

```
原始 M2305 收盘 3400
原始 M2309 开盘 3500  ← 切换日
差值 +100

前复权后：
  把 M2305 时段所有历史价格 +100
  → 历史 3400 调整为 3500
  保证切换日价格连续，无跳空
```

**优点**：得到一条干净的"价格序列"，可以直接画图、算技术指标
**缺点**：历史绝对价格被改了（10 年前的 3000 元在前复权图上可能显示成 4500）

### 方法 C：收益率拼接（returns-based stitching）—— 我们的选择

**核心思想**：不复权价格，复权**收益率**。

```python
对每一天 t:
    主力合约_t = 当天持仓量最大的合约
    主力合约_{t-1} = 昨天持仓量最大的合约

    if 主力合约_t == 主力合约_{t-1}:
        clean_return_t = (close_t - close_{t-1}) / close_{t-1}    ← 同一合约内的收益
    else:
        clean_return_t = NaN    ← 换月日，今天不计算收益
```

**为什么换月日是 NaN，不是 0**：
- 0 = "今天持仓没有盈亏"（错，因为我们手上的合约换了，无法对比）
- NaN = "今天没有可计算的收益"（对，更诚实）

**优点**：
- 直接得到可回测的收益率序列
- 数学上等价于完美的前复权（在收益率层面）
- 实现极简（不需要做价格平移的累计计算）

**缺点**：
- 没有"价格"概念了，只能画 NAV 曲线（cumprod(1+r)）
- 画 K 线、算价格相关的技术指标（如均线、布林带的绝对价位）需要另外保存原始合约价格

**结论**：**回测信号生成、PnL 计算用 clean returns；可视化或参考价格水平时用前复权或当时的实际合约价格**。

## 三、Hard Roll vs Adjusted：实证识别方法

### 如何判断手上的连续数据是 hard-spliced 还是已复权？

**方法 1：看换月日的极端收益**

```python
returns = continuous_close.pct_change()
extreme_days = returns.abs() > 0.05  # 5% 以上单日变动
print(returns[extreme_days])
```

- 如果出现密集的 |return| > 5% 但没有对应的市场新闻 → **hard-spliced**
- 如果只在真实市场冲击日出现 → **已复权**

**方法 2：与单合约对比**

把"M0 连续"和同期"M2309、M2401 单合约"画在一张图上：
- 如果 M0 在某天突然从 M2309 的轨迹"跳"到 M2401 的轨迹 → **hard-spliced**
- 如果 M0 平滑过渡 → **已复权**

→ 完整诊断脚本：`Phase_2_0_universe/03_verify_roll_logic.py`

### Sina/AkShare 现状

经我们实测：**Sina 的 PREFIX0 连续合约（如 M0、CU0）全部是 hard-spliced**，必须自己做收益率拼接或买已复权数据。

## 四、数据源选择决策框架

| 数据源 | 复权状态 | 成本 | 用途 |
|--------|---------|------|------|
| Sina via AkShare | Hard-spliced（无复权） | 免费 | 学习、单品种探索 |
| **自己做收益率拼接** | 等价于完美前复权（returns 层面） | 免费 + 工程时间 | 学习算法本身，POC |
| **Tushare Pro fut_daily** | 已复权连续合约 | ¥500/年起 | 生产 backtest |
| RQData / WindQuant | 已复权 + tick | ¥10k+/年 | 机构级 |

**我们的路径**：
1. ✅ 用 AkShare 学完"假跳空 → 收益率拼接"全过程，理解原理
2. → 转 Tushare Pro 拿现成的高质量数据，省掉工程负担
3. 后续策略迭代基于 Tushare Pro 数据

**为什么不一直用自拼数据**：
- 维护负担：每个品种的活跃月份要自己定义、symbol 格式各交易所不同
- 数据完整性：单合约数据缺失会传染到 clean returns（缺一段就 NaN 一段）
- 限流问题：免费 API 大批量拉取容易被封禁
- ROI 倒挂：会员费 ¥500/年 < 自己花 1 周维护脚本的时间成本

## 五、本节自检

1. M0 在 2024-07-15 显示了 -3.2% 的单日收益。已知当天 M2409 主力合约实际只动了 -0.5%。这个差额是怎么产生的？
   <details><summary>答案</summary>那天是换月日。M0 序列从 M2405 切换到了 M2409，两个合约本身就有 -2.7% 的价差（Backwardation：远月比近月便宜）。这 -2.7% 是"假跳空"，不是真实收益。</details>

2. 如果一定要用 hard-spliced 连续合约做信号生成（比如均线穿越），有什么后果？
   <details><summary>答案</summary>换月日的假跳空会触发虚假的均线穿越信号 → 系统在换月日"被迫"开/平仓 → 实际市场没动你却交易了 → 平白支付手续费和滑点。严重时会让一个本来盈利的策略亏钱。</details>

3. 为什么收益率拼接换月日要置 NaN，不是 0 也不是直接跳过？
   <details><summary>答案</summary>NaN 在 NumPy/Pandas 中会被聚合函数（mean、std）正确忽略，回测引擎也会跳过。而设 0 会被当成"今天平仓"的真实数据点，影响 vol 估计；直接跳过（删除整行）会破坏日期对齐，导致和其他品种 join 时索引混乱。</details>
